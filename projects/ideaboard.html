Ideaboard

A case study

This is how I designed a note-taking app for visual thinkers.

Overview

The gist

You are online and you find an interesting image for future reference. How do you save it? Do you create a google doc with a number of links? You cannot preview them, so when you try to access it- it’s just a sea of links! How frustrating. Sure, you can use Evernote, or Dropbox Paper or Pintrest… but what if you want to keep on adding links, move images around, compare them, change sizes, put text on it? Currently, creatives have to use multiple applications to keep their ideas. Idea Board is a one stop solution for all those needs.

The Problem

The biggest issue of this project that there were no specific goals on the onset of the project.

"Overall we think there's a lot of interesting and compelling features we could put into an app like this, but we don't have a clear vision for the project yet. We don't have a name or brand. We aren't sure if this should just be a web app for desktop browsers or if it should be usable on mobile devices too. We don't even have a target audience in mind really. We really need your thoughts on all this as we see an opportunity here that plays to our core strengths, but we need you to help us bring it to market. We can't wait to find out what your research unearths about this space!"

When we think about Cloud Storage, some big names immediately come to mind, including Google, Dropbox, Evernote and Microsoft.

The stakeholders had mentioned a number of possible features, including, creating content in various formats, saving content they find online, organizing content, sharing documents, collaborating with teammates as well as social sharing.

The problem of choice

Where is the biggest opportunity?

The paradox of choice is real. We were dealt with the problem of finding the problem - which is often harder than solving a well defined issue.

How do we best serve our customers?

Once we find that space to concentrate on, how do we create the best experience for our users? How do we stand out?

Our Process

Research

Creating a product without having an end-user in mind is not only hard, but it actually defeats the purpose. There are plenty of ways we can go about it, so how do we narrow the focus? We surveyed potential users, as well as did a competitive analysis of companies in similar space in order to figure out our approach.

User Survey

“A problem well stated is a problem half solved.” Charles Kettering 

I created a questionnaire to determine the core users as well as the features that they would actually want. My goal was to find under-served users so there’s potential to grow in the crowded market of cloud storage and organization.

I deployed a user survey to gather data in order to look for patterns, trends and outliers.

After analyzing the data that was collected, it was clear that the target user is a graphic designer or a student between the ages of 22-39. Most of these people are using Google Drive or Dropbox at the moment, however, they need to use multiple apps to do tasks like saving links they find online. They are also looking for more flexible options for creating documents.

Survey Analysis

Competitive Analysis

Cloud storage is a crowded market. There are a number of big-name competitors in the space. To beat them, we first have to know them.

I decided to take a deeper dive into Google Drive, Dropbox and Evernote to examine the positioning, primary audience(s), and their differentiators. My goal was to figure out we can identify why the competition has taken a particular course of action. The more we learn about the current market and industry standards, the more it will help define our product.

The Opportunity

There are riches in the niches

There is an under served market of Creative professionals. Visual designers, motion graphics artists, producers, directors, editors, etc. rely heavily on cloud because they need more storage than most people in other industries. The survey showed that most of the creatives also pay for additional storage and premium features on apps like Dropbox and Evernote. If we create collaborative cloud keeping in mind the needs of a specific group, we can provide a lot of value to our niche users and do not have to compete with the established giants

User Stories

To create a tool that would make it easy for users to perform the tasks that they do, I first need to understand their workflow. Most of our target market of creatives goes through the following steps.

  1. Write notes based on client conversation or creative brief
  2. Find inspiration on Behance, Pintrest, Dribble
  3. Create a moodboard by using screengrabs, downloaded assets, pasting video links. This is done in various platforms because currently, there’s no one tool that does all.
  4. Design.
  5. Save it to the cloud.
  6. Share it with colleagues.
  7. Send emails for iterations, along with links to the updated work on Dropbox, Google Drive or One Drive.

Using multiple tools to perform these tasks is not only time-consuming, but elements also get lost in translation. Moreover, when it comes to exploring options, seeing the big picture and connecting the dots, the available tools feel quite rigid and limiting. There are new platforms like Milanote that offer infinite canvas, but it is subpar when it comes to organizing notes.

The goal is to create a tool that allows for saving, organizing, storing and collaborating.

User Flows

I transformed my user stories into user flows. The goal was clear:

  1. I wanted to make note taking easy.
  2. All features had to be accessible and intuitive.
  3. Tabs, menus, tools had to be out of the way. The most important content is what user generates. So I wanted to provide a wide enough canvas with the tools and navigation as something that aids them, without distracting.
  4. The target market for this platform is extremely web savvy. Most of them would also be well versed in a few design/aesthetics related applications.
  5. Focus on making search quick and creating content easy.
